Servlet

サーブレット

Webサーバー上で動作するJavaプログラム。 ブラウザからリクエストを受け取り、 必要な処理を行ってレスポンスを返す。

試験ポイント： 通常はHttpServletを継承して作成する。

HttpServlet

エイチティーティーピー・サーブレット

HTTP通信を扱うServletの基本クラス。 作成するServletクラスで継承する。

基本形： public class Sample extends HttpServlet

@WebServlet

ウェブ・サーブレット

ServletクラスとURLパターンを対応させる アノテーション。

例： @WebServlet("/Login")

doGet()

ドゥー・ゲット

GETリクエストを受けたときに実行される Servletのメソッド。

検索・ページ表示・リンクからのアクセスなどで GETが使われることが多い。

doPost()

ドゥー・ポスト

POSTリクエストを受けたときに実行される Servletのメソッド。

HTMLフォームでは method="post"を指定する。

GET

ゲット・リクエスト

主に情報を取得するときに使われる リクエストメソッド。 送信データがURLへ付加される。

URLへデータが表示される点をPOSTと比較する。

POST

ポスト・リクエスト

主に入力内容の登録や送信に使われる リクエストメソッド。 データは主にリクエストボディへ格納される。

URLへデータが直接表示されない。 ただし、POSTだけで安全になるわけではない。

リクエスト

HTTP REQUEST

ブラウザなどのクライアントから、 Webサーバーへ送られる要求。

Servletでは通常、 HttpServletRequest request で扱う。

レスポンス

HTTP RESPONSE

Webサーバーが処理結果として、 ブラウザなどへ返す応答。

Servletでは通常、 HttpServletResponse response で扱う。

getParameter()

リクエストパラメータ取得

フォームやURLから送信された リクエストパラメータを取得するメソッド。

request.getParameter("name")
戻り値は基本的にString。

Content-Type

コンテント・タイプ

レスポンスで返すデータの種類や 文字コードをブラウザへ伝える情報。

response.setContentType("text/html; charset=UTF-8");

PrintWriter

プリント・ライター

Servletから文字列やHTMLを レスポンスへ出力するために使う。

PrintWriter out = response.getWriter();

JSP

JAVA SERVER PAGES

HTMLを中心に画面表示を作成する技術。 MVCでは主にViewを担当する。

Servletで大量のHTMLを生成するより、 表示部分を作りやすい。

スクリプトレット

SCRIPTLET

JSPファイル内へJavaの文や 制御構文を記述するための領域。

記号： <% Javaコード %>

スクリプト式

JSP EXPRESSION

Javaの式を評価し、その結果を JSPの画面へ出力する。

記号： <%= 式 %>
基本的に末尾へセミコロンを付けない。

JSPコメント

JSP COMMENT

JSP内で使用するコメント。 生成されるレスポンスHTMLには含まれない。

記号： <%-- コメント --%>

pageディレクティブ

PAGE DIRECTIVE

JSPページ全体に関する設定を行う。 Content-Typeや文字コードなどを指定できる。

<%@ page contentType="text/html; charset=UTF-8" %>

MVCモデル

MODEL・VIEW・CONTROLLER

アプリケーションをModel・View・Controllerの 3つの役割へ分ける設計方法。

Servlet＝Controller、JSP＝View、 JavaBeansなど＝Modelとして使われる。

Model

モデル

データや業務処理を担当する。 JavaBeansや処理用のJavaクラスなどが該当する。

「データと処理を担当」と説明できるようにする。

View

ビュー

利用者へ見せる画面表示を担当する。 ServletとJSPの構成では主にJSPが担当する。

Viewへ複雑な処理を書きすぎないことが基本。

Controller

コントローラー

リクエストを受け取り、 必要な処理や表示先を決定する。

ServletとJSPの構成では、 主にServletがControllerを担当する。

フォワード

FORWARD

サーバー内部で、別のServletやJSPへ リクエスト処理を転送する。

サーバー内部の転送なので、 ブラウザのURLは基本的に変わらない。

RequestDispatcher

リクエスト・ディスパッチャー

Servletから別のServletやJSPへ フォワードするときに利用する。

request.getRequestDispatcher("転送先")
dispatcher.forward(request, response)

リダイレクト

REDIRECT

ブラウザへ移動先を伝え、 その移動先へ新しいリクエストを送らせる。

response.sendRedirect("移動先")
ブラウザのURLは移動先へ変わる。

WEB-INF

ウェブ・インフ

Webアプリケーションの設定ファイルなどを 配置する特別なフォルダ。

JSPをWEB-INF内へ配置すると、 ブラウザからの直接アクセスを防げる。 Servletからフォワードして表示する。

スコープ

SCOPE

保存したデータを共有・利用できる範囲。 用途に応じて適切なスコープを選ぶ。

リクエスト・セッション・アプリケーションの 違いを説明できるようにする。

リクエストスコープ

REQUEST SCOPE

基本的に1回のリクエスト処理内で データを共有するスコープ。

Servletからフォワード先のJSPへ データを渡す場合に適している。

セッションスコープ

SESSION SCOPE

同じ利用者の複数リクエスト間で データを共有するスコープ。

ログイン情報や買い物かごなどで使われる。 不要になったデータは削除する。

アプリケーションスコープ

APPLICATION SCOPE

Webアプリケーション全体で データを共有するスコープ。

複数の利用者から共有される。 共有範囲が非常に広い。

setAttribute()

属性の保存

スコープへ名前を付けて オブジェクトを保存するメソッド。

request.setAttribute("user", user);

getAttribute()

属性の取得

スコープへ保存されたオブジェクトを 名前を指定して取得するメソッド。

request.getAttribute("user")
戻り値は基本的にObject。

JavaBeans

ジャバ・ビーンズ

関連するデータをプロパティとしてまとめて扱う Javaクラス。

プロパティは基本的にprivateとし、 getter・setterを使って操作する。

プロパティ

PROPERTY

JavaBeansが持つ名前や年齢などのデータ。 getterとsetterを通して操作する。

nameプロパティなら getName()と setName()を用意する。

getter

ゲッター

JavaBeansのプロパティの値を 取得するためのメソッド。

nameプロパティなら public String getName()

setter

セッター

JavaBeansのプロパティへ 値を設定するためのメソッド。

nameプロパティなら public void setName(String name)

Integer.parseInt()

文字列からintへの型変換

数字を表すStringをint型へ変換する。

int age = Integer.parseInt(ageText);
数字以外の場合は例外が発生する可能性がある。
