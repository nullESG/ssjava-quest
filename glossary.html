Servlet・JSP・MVC・フォワード・スコープなど、 試験範囲の重要語句をまとめています。 説明問題で書けるように、意味と違いを確認しましょう。

Servlet

サーブレット

Webサーバー上で動作するJavaプログラム。 ブラウザからリクエストを受け取り、 必要な処理を行ってレスポンスを返す。

試験ポイント： 通常はHttpServletを継承して作成する。

HttpServlet

エイチティーティーピー・サーブレット

HTTP通信を扱うServletの基本クラス。 作成するServletクラスで継承する。

基本形： public class Sample extends HttpServlet

@WebServlet

ウェブ・サーブレット

ServletクラスとURLパターンを対応させる アノテーション。

例： @WebServlet("/Login")

doGet()

ドゥー・ゲット

GETリクエストを受けたときに実行される Servletのメソッド。

検索・ページ表示・リンクからのアクセスなどで GETが使われることが多い。

doPost()

ドゥー・ポスト

POSTリクエストを受けたときに実行される Servletのメソッド。

HTMLフォームでは method="post"を指定する。

GET

ゲット・リクエスト

主に情報を取得するときに使われる リクエストメソッド。 送信データがURLへ付加される。

URLへデータが表示される点をPOSTと比較する。

POST

ポスト・リクエスト

主に入力内容の登録や送信に使われる リクエストメソッド。 データは主にリクエストボディへ格納される。

URLへデータが直接表示されない。 ただし、POSTだけで安全になるわけではない。

リクエスト

HTTP REQUEST

ブラウザなどのクライアントから、 Webサーバーへ送られる要求。

Servletでは通常、 HttpServletRequest request で扱う。

レスポンス

HTTP RESPONSE

Webサーバーが処理結果として、 ブラウザなどへ返す応答。

Servletでは通常、 HttpServletResponse response で扱う。

getParameter()

リクエストパラメータ取得

フォームやURLから送信された リクエストパラメータを取得するメソッド。

request.getParameter("name")
戻り値は基本的にString。

Content-Type

コンテント・タイプ

レスポンスで返すデータの種類や 文字コードをブラウザへ伝える情報。

response.setContentType("text/html; charset=UTF-8");

PrintWriter

プリント・ライター

Servletから文字列やHTMLを レスポンスへ出力するために使う。

PrintWriter out = response.getWriter();

JSP

JAVA SERVER PAGES

HTMLを中心に画面表示を作成する技術。 MVCでは主にViewを担当する。

Servletで大量のHTMLを生成するより、 表示部分を作りやすい。

スクリプトレット

SCRIPTLET

JSPファイル内へJavaの文や 制御構文を記述するための領域。

記号： <% Javaコード %>

スクリプト式

JSP EXPRESSION

Javaの式を評価し、その結果を JSPの画面へ出力する。

記号： <%= 式 %>
基本的に末尾へセミコロンを付けない。

JSPコメント

JSP COMMENT

JSP内で使用するコメント。 生成されるレスポンスHTMLには含まれない。

記号： <%-- コメント --%>

pageディレクティブ

PAGE DIRECTIVE

JSPページ全体に関する設定を行う。 Content-Typeや文字コードなどを指定できる。

<%@ page contentType="text/html; charset=UTF-8" %>

MVCモデル

MODEL・VIEW・CONTROLLER

アプリケーションをModel・View・Controllerの 3つの役割へ分ける設計方法。

Servlet＝Controller、JSP＝View、 JavaBeansなど＝Modelとして使われる。

Model

モデル

データや業務処理を担当する。 JavaBeansや処理用のJavaクラスなどが該当する。

「データと処理を担当」と説明できるようにする。

View

ビュー

利用者へ見せる画面表示を担当する。 ServletとJSPの構成では主にJSPが担当する。

Viewへ複雑な処理を書きすぎないことが基本。

Controller

コントローラー

リクエストを受け取り、 必要な処理や表示先を決定する。

ServletとJSPの構成では、 主にServletがControllerを担当する。

フォワード

FORWARD

サーバー内部で、別のServletやJSPへ リクエスト処理を転送する。

サーバー内部の転送なので、 ブラウザのURLは基本的に変わらない。

RequestDispatcher

リクエスト・ディスパッチャー

Servletから別のServletやJSPへ フォワードするときに利用する。

request.getRequestDispatcher("転送先")
dispatcher.forward(request, response)

リダイレクト

REDIRECT

ブラウザへ移動先を伝え、 その移動先へ新しいリクエストを送らせる。

response.sendRedirect("移動先")
ブラウザのURLは移動先へ変わる。

WEB-INF

ウェブ・インフ

Webアプリケーションの設定ファイルなどを 配置する特別なフォルダ。

JSPをWEB-INF内へ配置すると、 ブラウザからの直接アクセスを防げる。 Servletからフォワードして表示する。

スコープ

SCOPE

保存したデータを共有・利用できる範囲。 今回の試験では、ServletからJSPへデータを渡すための リクエストスコープが重要。

ServletでsetAttribute()を使って保存し、 フォワード先のJSPでgetAttribute()を使って 取得する流れを覚える。

リクエストスコープ

REQUEST SCOPE

1回のリクエスト処理内でデータを共有するスコープ。 Servletからフォワード先のJSPへ、 JavaBeansなどのインスタンスを渡すときに使用する。

保存： request.setAttribute("user", user);
取得： request.getAttribute("user");
リクエストスコープのデータをJSPへ渡す場合は、 同じリクエストを渡すフォワードを使う。

setAttribute()

属性の保存

スコープへ名前を付けて オブジェクトを保存するメソッド。

request.setAttribute("user", user);

getAttribute()

属性の取得

スコープへ保存されたオブジェクトを 名前を指定して取得するメソッド。

request.getAttribute("user")
戻り値は基本的にObject。

JavaBeans

ジャバ・ビーンズ

関連するデータをプロパティとしてまとめて扱う Javaクラス。

プロパティは基本的にprivateとし、 getter・setterを使って操作する。

プロパティ

PROPERTY

JavaBeansが持つ名前や年齢などのデータ。 getterとsetterを通して操作する。

nameプロパティなら getName()と setName()を用意する。

getter

ゲッター

JavaBeansのプロパティの値を 取得するためのメソッド。

nameプロパティなら public String getName()

setter

セッター

JavaBeansのプロパティへ 値を設定するためのメソッド。

nameプロパティなら public void setName(String name)

Integer.parseInt()

文字列からintへの型変換

数字を表すStringをint型へ変換する。

int age = Integer.parseInt(ageText);
数字以外の場合は例外が発生する可能性がある。

form要素

HTML FORM

利用者が入力したデータをServletなどへ 送信するためのHTML要素。

actionに送信先、 methodにGETまたはPOSTを指定する。
例： <form action="Register" method="post">

action属性

FORM ACTION

フォームデータの送信先を指定する属性。 ServletのURLパターンなどを指定する。

例： action="Register"
@WebServlet("/Register")のServletと 対応させる。

method属性

FORM METHOD

フォームをGETとPOSTのどちらで送信するか指定する属性。

GET送信： method="get"
POST送信： method="post"

name属性

FORM CONTROL NAME

フォーム部品の送信データへ名前を付ける属性。 リクエストパラメータ名として使われる。

HTML： <input name="age">
Java： request.getParameter("age")
両方の名前を一致させる。

アンカー

ANCHOR・LINK

HTMLの<a>要素。 リンクからServletへアクセスすると、 基本的にGETリクエストになる。

例： <a href="Menu">メニューへ</a>

クエリ文字列

QUERY STRING

GETリクエストで、URLの末尾へ付加される パラメータ部分。

例： Detail?id=10&mode=edit
最初のパラメータは?、 2つ目以降は&で区切る。

URLパターン

URL PATTERN

ブラウザからのURLと、 実行するServletクラスを対応させるための指定。

@WebServlet("/Login")なら、 WebアプリケーションのURLに続けて /Loginへアクセスする。

オーバーライド

OVERRIDE

親クラスから継承したメソッドを、 子クラス側で同じ形式で定義し直すこと。

HttpServletを継承したクラスで、 doGet()やdoPost()を オーバーライドする。
@Overrideを付けることが多い。

getParameterとgetAttributeの違い

PARAMETER VS ATTRIBUTE

getParameter()はフォームやURLから 送信された値を取得する。 getAttribute()はスコープへ保存した オブジェクトを取得する。

送信値： request.getParameter("age")
保存済み属性： request.getAttribute("user")

getParameterの戻り値

STRING VALUE

request.getParameter()で取得した値は、 基本的にString型になる。

数値として計算する場合は型変換する。
int age = Integer.parseInt(request.getParameter("age"));

フォワードの基本構文

FORWARD SYNTAX

RequestDispatcherを取得し、 requestとresponseを転送する。

RequestDispatcher dispatcher = request.getRequestDispatcher( "/WEB-INF/jsp/result.jsp");
dispatcher.forward(request, response);

フォワードとリダイレクトの違い

FORWARD VS REDIRECT

フォワードはサーバー内部の転送。 リダイレクトはブラウザへ再リクエストさせる処理。

フォワード：URLは基本的に変わらない。
リダイレクト：移動先のURLへ変わる。
リクエストスコープをJSPへ渡す場合は フォワードが適している。

JSPの実行

JSP EXECUTION

JSPはWebサーバー上で処理され、 最終的にHTMLなどのレスポンスを返す。

MVC構成では、ブラウザからJSPへ直接アクセスするより、 Servletで処理してからJSPへフォワードする形が基本。

JSPの必要性

WHY JSP?

Servletでprintln()を使って 大量のHTMLを生成すると、 Javaコードと表示コードが混ざって複雑になる。

JSPを使うとHTMLを中心に画面を作成できる。 MVCではJSPをViewとして利用する。

MVCの処理の流れ

MVC FLOW

Servletがリクエストを受け、 必要な処理やデータ作成を行い、 JSPへフォワードして画面を表示する。

基本の流れ：
ブラウザ
↓ リクエスト
Servlet（Controller）
↓ JavaBeansをスコープへ保存
JSP（View）
↓ レスポンス
ブラウザ

インスタンスをスコープへ保存

STORE AN INSTANCE

ServletでJavaBeansなどのインスタンスを作成し、 リクエストスコープへ保存する。

User user = new User();
user.setName(name);
request.setAttribute("user", user);

キャスト

CAST

getAttribute()の戻り値は基本的に Object型なので、 元の型として扱う場合に型変換する。

User user = (User) request.getAttribute("user");
